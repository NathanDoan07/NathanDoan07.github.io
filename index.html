Rust Lexical Analyzer + Syntax Parser

A solo project to construct a lexical analyzer and syntax parser for a given language and its grammar written in Rust.

The program can take in a file and look for lexical errors in the given grammar. It then creates tokens for the analyzer and parses through the given language.

The lexer stores tokens using a Regex expression to look for certain keywords that are given or are required in the grammar. The analyzer parses through each token and checks if it adheres to the given grammar of the program.

The program also uses flags for output in Scheme or Prolog.

GitHub Repo: Rust

Webcrawler - Graph GUI

A group project with 2 other students to create a graph GUI for analyzing and visualizing the closeness centrality of a given website. Closeness centrality is a measure on how close a website node is to the given starting websites.

These students were: Daniel Adebayoo, Reagan

The project takes in a file of a starting website URl, then based on the given depth will look to continue to chain through websites using Dijkstra's algorithm that the given website is connected to.

My role in the project was creating the GUI element of the graph and helping collaboratively on the process in creating the connecting nodes of the websites together. Using a variety of Python libraries such as NetworkX and Matplotlibs, the process of creating the graph was team effort, but more towards my responsibility.
